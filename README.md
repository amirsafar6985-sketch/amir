# 🎧 Amir Player — ساختار قسمت‌بندی‌شده

کد اصلی که یک فایل تک‌پارچه‌ی ۱۶۰۰+ خطی بود، به فایل‌های جدا بر اساس
قابلیت تقسیم شد. **`main.py`** تنها فایلیه که پنجره رو می‌سازه، ماژول‌های
دیگه رو به هم وصل می‌کنه و حلقه‌های اصلی اجرایی (mainloop و حلقه‌ی
به‌روزرسانی پیشرفت پخش) رو اجرا می‌کنه.

## اجرای برنامه
```
pip install -r requirements.txt
python main.py
```

## ساختار فایل‌ها

| فایل | مسئولیت |
|---|---|
| `main.py` | نقطه ورود؛ ساخت پنجره، اتصال ماژول‌ها، اجرای `mainloop()` و حلقه‌ی `update_progress()` |
| `state.py` | یک شیء `state` واحد که همه‌ی متغیرهای مشترک (پلی‌لیست‌ها، وضعیت پخش، رفرنس ویجت‌ها و ...) رو نگه می‌داره؛ جایگزین `global` های پراکنده |
| `constants.py` | رنگ‌ها، پریست‌های اکولایزر، پریست‌های گرادیان پس‌زمینه |
| `utils.py` | `debounce`, `safe_ui`, `format_time` |
| `settings_manager.py` | ذخیره/بازیابی تنظیمات از/به `player_settings.json` |
| `background.py` | ساخت و رندر گرادیان یا عکس دلخواه پس‌زمینه |
| `theme.py` | رنگ‌های تم روشن/تاریک و اعمال تم به ویجت‌ها |
| `animations.py` | انیمیشن تغییر رنگ نرم، اسلاید پنل، پالس دکمه |
| `audio_engine.py` | پخش/توقف/ادامه، آهنگ بعدی-قبلی، اکولایزر VLC، اسلایدر پیشرفت |
| `playlist_manager.py` | افزودن/حذف آهنگ، کارت‌های UI، درگ‌اند‌دراپ، مدیریت پلی‌لیست‌ها |
| `platform_utils.py` | افکت شیشه‌ای ویندوز ۱۱ (Mica) |
| `ui/player_bar.py` | ساخت نوار پخش پایین صفحه (نام آهنگ، اسلایدر، دکمه‌ها، صدا) |
| `ui/settings_panel.py` | ساخت پنل تنظیمات کناری (پلی‌لیست، سوییچ‌ها، پس‌زمینه، اکولایزر) |

## نکته مهم درباره‌ی state.py
برای این که چند فایل بتونن مثل قبل به همون متغیرهای مشترک (`current_index`,
`is_playing`, ویجت‌ها و ...) دسترسی داشته باشن، همه‌شون رو داخل یک شیء
`AppState` جمع کردم و همه‌ی ماژول‌ها به جای `global x`، از `state.x`
استفاده می‌کنن. این باعث میشه رفتار برنامه دقیقاً مثل نسخه‌ی تک‌فایلی
اصلی بمونه، ولی کد بین چند فایل قابل نگهداری‌تر تقسیم بشه.

هیچ منطقی حذف یا تغییر داده نشده؛ فقط جابه‌جایی و سازمان‌دهی مجدد کد انجام
شده (همه‌ی توابع اصلی با همون اسم قبلی موجودن).
